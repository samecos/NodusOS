# Nodus 产品需求文档 (PRD)

> 版本: v0.1 | 状态: Draft | 日期: 2026-05-04

---

## 1. 产品概述

### 1.1 产品愿景

Nodus（结绳）是一个AI原生操作系统。传统OS是"硬件资源调度器"——管理者是CPU、内存、硬盘，人是操作员。Nodus是"人类意图与认知资源调度器"——核心路由是大语言模型，人是指挥官。

编程不是操作系统上的一个应用。编程是操作系统存在的理由。

### 1.2 一句话定义

用说话的方式理解代码库。环境这件事不让人看见。

### 1.3 核心价值主张

| 维度 | 传统OS | Nodus |
|------|--------|-------|
| 工作单位 | 文件与应用 | 意图与服务 |
| 交互逻辑 | 人找工具（点击、菜单、快捷键） | 工具找人（自然表达，后台组装） |
| 人的角色 | 操作员（跨软件搬运工） | 指挥官（提目标、审结果） |
| 学习成本 | 人迁就机器（学Excel函数、背快捷键） | 机器迁就人（适应用户口语和偏好） |
| 容错率 | 零容错（敲错就卡死） | 高容错（模糊时反问澄清） |

## 2. 目标用户

### 2.1 首批用户：开发者（Early Adopter / Programmer）

Nodus的首批目标用户是**程序员**。原因：
1. 作者自己是程序员（"吃自己的狗粮"）
2. 开发者场景复杂度最高，对系统骨架的健壮性要求最强
3. 开发者是互联网上传播力最强的人群
4. 为后续Skill生态奠定贡献者基础

### 2.2 开发者核心痛点

| 痛点 | 现状 | Nodus解法 |
|------|------|-----------|
| 上下文重建 | 每天花10-15分钟恢复昨天的工作状态 | 系统记住上下文，主动呈现 |
| 代码库导航 | grep + 经验 + 问同事，30分钟画调用图 | 语义理解代码库，秒级定位 |
| 跨域调试 | 日志、代码、git分散在多个工具中，脑内关联 | 系统自动关联多维度信息 |
| 环境搭建 | 2-4小时配置开发环境，每个新设备重复 | 零配置，自动检测并安装一切 |
| 信息碎片化 | 编辑器/终端/浏览器/Slack之间搬信息 | 系统当信息集成者 |

### 2.3 后续延伸用户

- **知识工作者** (v2)：研究员、产品经理、创业者——痛点同构（信息碎片化+上下文切换），只需接入不同的能力组件
- **数字难民** (v3)：老年人、技术恐惧者——同一引擎换更简单的语音皮肤

## 3. 产品核心原则

### 3.1 六项原则

| # | 原则 | 说明 |
|---|------|------|
| P1 | 意图驱动，非指令驱动 | 用户表达What，系统自主铺路How |
| P2 | 语音优先，界面为辅 | 闭眼完成80%操作；UI只是辅助通道 |
| P3 | 系统适应人 | 无学习成本；老奶奶和高级工程师同一套系统 |
| P4 | 全局上下文，无边界调度 | 打破App数据孤岛，统一语义空间 |
| P5 | 主动计算，人授权限 | 按领域独立授权；用户始终是决策者 |
| P6 | 认知卸载，务实降级 | 宏观完全卸载，微观诚实降级 |

### 3.2 反原则（明确不做的事）

- 不是AI加壳的传统桌面（不是Copilot in Windows）
- 不是聊天机器人界面（ChatGPT套壳）
- 不要求用户理解计算机概念（文件路径、App名、菜单层级）
- 不是通知轰炸机（主动推送必须过三道关）
- 不是封闭花园（开放Skill生态）
- 不假装自己全能（做不到的诚实说）

### 3.3 原则优先级（冲突时裁决顺序）

1. 人授权限 > 主动计算
2. 务实降级 > 体验一致性
3. 语音优先 > 视觉呈现
4. 系统适应人 > 功能完整性

## 5. 非功能需求 (Non-Functional Requirements)

### 5.1 性能

| ID | 指标 | 目标值 | 测量方式 |
|----|------|--------|---------|
| NFR1 | 意图解析延迟 | < 200ms | 从语音转写完成到QueryIntent生成 |
| NFR2 | 查询响应延迟 | < 3s | 从QueryIntent到卡片显示（索引已就绪） |
| NFR3 | 首次索引时间 | < 30s (1万行项目) | 从项目打开到索引就绪 |
| NFR4 | 环境就绪时间 | < 2min | 从项目打开到环境完全就绪 |
| NFR5 | 上下文快照延迟 | < 1ms | ContextManager.snapshot() 调用耗时 |

### 5.2 可靠性

| ID | 指标 | 目标值 |
|----|------|--------|
| NFR6 | 符号索引覆盖率 | ≥ 95%（10万行以内项目） |
| NFR7 | 调用图准确率 | ≥ 95% |
| NFR8 | 增量索引一致性 | 文件修改后5秒内索引更新 |
| NFR9 | 单文件解析失败影响 | 不影响其他文件的索引和查询 |

### 5.3 可用性

| ID | 指标 | 目标值 |
|----|------|--------|
| NFR10 | 无需阅读文档可完成首次查询 | 是（语音引导或自然语言即可） |
| NFR11 | 歧义反问准确率 | 歧义时100%反问，不猜测 |
| NFR12 | 无声模式支持 | 完整文字交互，不依赖语音 |

### 5.4 兼容性

| ID | 需求 | 说明 |
|----|------|------|
| NFR13 | 操作系统 | MVP仅macOS（桌面端先） |
| NFR14 | 语言支持 | JS、TS、Python（架构语言无关，插件扩展） |
| NFR15 | 本地优先 | MVP不依赖任何云服务 |
| NFR16 | 不替代现有编辑器 | 与VSCode等共存，Nodus是OS层信息集成者 |

## 6. MVP 范围定义

### 6.1 MVP 包含

1. 打开项目自动检测语言并安装环境（JS/TS/Python）
2. 代码库语义索引（符号、引用、调用图、变更历史）
3. 六类自然语言查询（定义定位/引用查找/调用链路/影响分析/变更历史/符号概览）
4. 语音输入（唤醒词+转写+解析）和文字输入（Ctrl+Space）
5. 结构化结果卡片（调用图/引用列表/变更历史/环境状态）
6. 代码跳转查看

### 6.2 MVP 明确不做

- AI代码生成/重构 → v2
- 跨域调试（日志+代码关联） → v2
- 自动标注训练飞轮 → v2
- 多设备同步 → v2
- JS/TS/Python以外的语言 → v2
- 团队协作 → v2
- 主动计算/通知推送 → v2
- 外部服务环境（DB/Redis等） → v2

### 6.3 MVP 成功标准

1. **环境就绪**：标准JS/TS/Python项目，2分钟内环境就绪，无人工命令
2. **索引准确**：10万行项目，符号索引覆盖率≥95%，调用图准确率≥95%
3. **查询体验**：索引就绪后，意图到结果卡片不超过3秒
4. **生存测试**：作者自己日常使用Nodus开发，持续一周——如果自己都不想用，方向错了

## 7. 用户故事 (User Stories)

### US1: 早晨恢复
> 作为开发者，我打开Nodus后，系统立即告诉我昨天写到哪里了、测试结果如何，无需我手动翻git log和终端历史。

**验收条件**：打开项目后5秒内显示上次会话的上下文（最后编辑的文件、光标位置、未提交更改、测试结果）。

### US2: 代码库导航
> 作为开发者，我想知道"refundOrder这个函数被哪些地方调用了"，不需要grep后逐行看127行输出。

**验收条件**：语音查询后3秒内展示结构化引用列表，标注潜在风险。

### US3: 环境零配置
> 作为开发者，我克隆一个新项目后，系统自动安装正确的运行时和依赖，我不用输入任何命令。

**验收条件**：标准项目从打开到就绪不超过2分钟，全程无人工命令输入。

### US4: 无声模式切换
> 作为开发者在开放办公室，我按Ctrl+Space输入文字意图，系统以卡片回应，不发出声音。

**验收条件**：无声模式下唤醒词监听停止，文字输入完整可用。

### US5: 模糊意图反问
> 当我说"把那个改一下"，系统不确定我指什么，生成几个候选让我选，而不是瞎猜。

**验收条件**：歧义时100%反问，不猜测。卡片提供2-3个候选选项。

## 8. 术语表

| 术语 | 定义 |
|------|------|
| Intent | 用户意图——自然语言表达的"要什么" |
| QueryIntent | 结构化查询意图——NLU解析后的机器可读结构 |
| Card | 结果卡片——按需生成、用完即焚的UI单元 |
| Skill/能力组件 | 系统可调用的后台能力（类比传统App，但对用户不可见） |
| Symbol | 代码符号——函数、类、接口、变量等的统称 |
| CallGraph | 调用图——符号间调用关系的图结构 |
| Semantic Index | 语义索引——代码库的符号、引用、调用图的总和 |
| Breath Light | 呼吸灯——屏幕边缘的微弱状态指示器 |
| Silent Mode | 无声模式——停止语音监听，仅文字交互 |
| Wake Word | 唤醒词——"Nodus"/"结绳" |
