# 目标用户画像

## 三类核心用户

### 类型A：数字难民（Silver Surfer）
**典型画像**：60岁以上的退休人士、技术恐惧者、残障人士。有生活需求（购物、社交、看病预约），但被现有数字工具排斥在外。

| 维度 | 描述 |
|------|------|
| **核心痛点** | 图标看不懂、菜单找不到、怕点错、子女教完就忘 |
| **当前解决方案** | 求助子女/邻居，或干脆不去用 |
| **关键使用场景** | 网购火车票、医院挂号、和孙辈视频、看新闻 |
| **对Nodus的价值** | 语音优先 + 系统适应人 = 这条线最直接的受益者 |
| **可规模化性** | 极高（中国2.8亿60岁以上人口），但商业转化路径长 |
| **Nodus适配度** | ★★★★★（天然匹配） |

**一句话**：只要会说话就能用电脑——这是他们人生中第一次。

### 类型B：知识工作者（Knowledge Worker）
**典型画像**：25-45岁的研究员、产品经理、创业者、高管。每天在多个工具间搬运信息，脑力大量浪费在"数字体力活"上。

| 维度 | 描述 |
|------|------|
| **核心痛点** | 信息碎片化、跨App搬运疲劳、注意力被通知撕裂、格式化劳动（PPT/邮件/报表）占据大量时间 |
| **当前解决方案** | 自己学快捷键、雇助理、忍受低效 |
| **关键使用场景** | 会议准备（检索+整理+做材料）、跨渠道信息整合、日程管理、文档撰写 |
| **对Nodus的价值** | 认知卸载最直接的体验者，"省下脑力做创造性工作"的强烈需求 |
| **可规模化性** | 高（全球知识工作者数亿），付费意愿强，商业路径清晰 |
| **Nodus适配度** | ★★★★☆（需要能力组件生态较丰富后才能充分体现价值） |

**一句话**：终于可以把"操作电脑"的脑力，还给"思考问题"本身。

### 类型C：极客先锋（Early Adopter）
**典型画像**：20-35岁的开发者、独立创作者、技术爱好者。享受效率工具，但厌倦了碎片化的工具链。

| 维度 | 描述 |
|------|------|
| **核心痛点** | 工具太多、配置太累、上下文在工具间断裂、重复性操作想自动化但懒得写脚本 |
| **当前解决方案** | 自己写脚本/插件、折腾各种效率工具 |
| **关键使用场景** | 开发工作流自动化、信息聚合、快速原型、跨设备协同 |
| **对Nodus的价值** | 会是最早的传播者，也是Skill生态的第一批贡献者 |
| **可规模化性** | 中（人群规模小），但影响力高（KOL效应） |
| **Nodus适配度** | ★★★☆☆（他们的需求更偏向"可编程性"而非"完全托管"） |

**一句话**：最好的工具不是更多的工具，而是一个能听懂我要什么的系统。

## MVP阶段的核心抉择

三类用户的需求**有本质差异**，不可能在一个MVP里同时完美服务：

| 冲突维度 | A-数字难民 | B-知识工作者 | C-极客先锋 |
|----------|-----------|-------------|-----------|
| 交互复杂度容忍度 | 零（必须极简） | 中（接受一定复杂度） | 高（甚至想要控制权） |
| 语音占比 | 几乎100% | 50-70% | 30-50% |
| 对可编程性需求 | 零 | 低 | 高 |
| 能力组件丰富度要求 | 低（几个核心场景） | 高（需要跨工具链） | 中 |
| 付费意愿 | 低 | 高 | 中 |
| 传播动力 | 弱 | 中 | 强 |

**最关键的矛盾**：A类用户需要"少到极致的功能，做到极致简单"；B类用户需要"足够多的能力连接，做到足够智能"；C类用户需要"开放可编程，别替我乱做主"。这三个方向指向三套不同的MVP。

---

## 推荐：以A（数字难民）为切入点，向B（知识工作者）延伸

理由：

1. **原则对齐度最高**。"语音优先""系统适应人""认知卸载"这三个核心原则，在A类用户身上体现得最纯粹。如果连最怕技术的人都能用，那其他人自然也能用。

2. **验证价值最大**。A类用户是"最难骗"的用户群——他们不会因为"AI很酷"就容忍糟糕体验。如果Nodus能让一个70岁老太太成功买到火车票，这个产品就真的立住了。

3. **差异化最强**。市面上所有"AI助手"都在卷知识工作者的效率（Copilot、Notion AI、ChatGPT），没人认真为被数字时代抛弃的人做操作系统。这是一片蓝海。

4. **向B延伸路径清晰**。一旦在A类用户上验证了"意图驱动+语音优先+务实降级"的核心骨架，向B类用户延伸只是增加更多能力组件和场景覆盖的问题。

**风险提示**：A类用户的商业转化路径较长。如果团队需要快速证明商业价值，可能需要A+B混合策略——用A类场景定交互基调，用B类场景证明付费价值。
