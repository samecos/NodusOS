# R2.3 查询历史与推荐 + R2.5 呼吸灯与状态指示

> 日期: 2026-07-02 | 状态: Approved | 关联: `05-Future-Roadmap.md` R2.3 / R2.5

---

## 1. 目标

### R2.3 查询历史与推荐（C 层）

- `/history` 命令：显示最近 N 条查询记录。
- 查询缓存：同一查询（规范化文本 + 上下文哈希）在 TTL 窗口内直接返回缓存结果。
- 主动推荐：用户输入空行时，基于最近查询、高频查询和当前上下文生成 ≤ 3 条推荐，用户输入序号即可执行。

### R2.5 呼吸灯与状态指示

- 在终端 REPL 中以带颜色与图标的单行状态指示替代当前的纯文本 `[thinking]` 标签。
- `NodusShell` 在查询、错误、语音等关键生命周期点自动切换状态。

## 2. 架构

```
main.ts (REPL)
  │
  ├── /history ──► NodusShell.getHistory(limit) ──► TerminalRenderer.renderHistory
  ├── empty line ──► NodusShell.getRecommendations() ──► RecommendationEngine
  │                                             └──► TerminalRenderer.renderRecommendations
  │
  ├── query ──► NodusShell.handleQueryFormatted
  │              ├── setBreathLight('thinking')
  │              ├── QueryCache.get(text+ctxHash)
  │              │     ├── hit  ──► return cached + setBreathLight('idle')
  │              │     └── miss ──► IntentEngine.parse → CodeIntel.query
  │              │                   ├── QueryCache.set(result)
  │              │                   └── setBreathLight('idle')
  │              └── error ──► setBreathLight('error')
  │
  └── /quit
```

## 3. 组件设计

### 3.1 QueryCache (`src/shell/query-cache.ts`)

```typescript
interface CacheEntry {
  result: string;          // handleQueryFormatted 的输出字符串
  createdAt: number;
}

export class QueryCache {
  constructor(maxSize: number = 50, ttlMs: number = 5 * 60 * 1000) {}
  get(key: string): string | null    // 过期则返回 null 并删除
  set(key: string, value: string): void
  clear(): void
  static buildKey(text: string, context: Context): string  // text + activeFile + cursorSymbol
}
```

- LRU 策略：超过 `maxSize` 时删除最旧条目。
- TTL 默认 5 分钟，过期自动失效。
- key = `normalizedText|file:symbol`，规范化为小写 + trim。

### 3.2 RecommendationEngine (`src/shell/recommendation-engine.ts`)

```typescript
export interface Recommendation {
  text: string;       // 可直接执行的查询文本
  reason: string;     // 推荐理由（用于展示）
}

export class RecommendationEngine {
  constructor(store: KnowledgeStore, contextMgr: ContextManager) {}

  /** 返回最多 3 条推荐 */
  generate(): Recommendation[]
}
```

推荐策略（按优先级）：

1. **上下文关联**（最高优先级）：若 `cursor_symbol` 存在，生成 "X被哪些地方调用了" 和 "如果我改了X，哪些文件会受影响"。
2. **高频查询**：从 `historyRecent(50)` 统计出现 ≥ 2 次的查询，取 top 2。
3. **最近查询延续**：从 `recent_queries[0]` 提取符号名，生成 "X的调用链路是什么样的"。

### 3.3 呼吸灯状态指示

#### 状态映射

| BreathLightState | 图标 | ANSI 颜色 | 触发点 |
|-----------------|------|----------|--------|
| `idle` | `○` | `\x1b[2m` (dim) | REPL 等待输入 / 查询完成 |
| `listening` | `◐` | `\x1b[36m` (cyan) | 语音管线就绪 |
| `thinking` | `◑` | `\x1b[33m` (yellow) | 查询解析/执行中 |
| `speaking` | `●` | `\x1b[32m` (green) | TTS 播放中 |
| `error` | `✖` | `\x1b[31m` (red) | 降级卡片触发 |

#### TerminalRenderer.setBreathLight

```
○ Nodus  // dim, idle
◑ thinking...  // yellow, thinking
✖ error  // red, error
```

格式：`{图标} {标签}`，带 ANSI 颜色。

#### NodusShell 集成点

- `bootstrap()` 结束 → `setBreathLight('idle')`
- `handleQueryFormatted()` 进入 → `setBreathLight('thinking')`
- `handleQueryFormatted()` 正常退出 → `setBreathLight('idle')`
- `handleQueryFormatted()` catch → `setBreathLight('error')`
- `error` 事件路由 → `setBreathLight('error')`
- `voicePipeline.start()` 后 → `setBreathLight('listening')`（仅非 silent 模式）
- TTS 播放前 → `setBreathLight('speaking')`，播放后 → `setBreathLight('idle')`

### 3.4 新增 EventBus 事件

```typescript
export interface UIStateChangedEvent {
  kind: 'ui:state_changed';
  state: BreathLightState;
}
```

加入 `NodusEvent` 联合类型。`TerminalRenderer.setBreathLight` 触发此事件（通过 NodusShell 转发）。

### 3.5 扩展 CardKind

新增两种卡片类型：

```typescript
| 'history_list'        // 查询历史列表
| 'recommendation_list' // 推荐列表
```

### 3.6 KnowledgeStore.historySearch

```typescript
/** 按关键词模糊搜索查询历史 */
historySearch(keyword: string, limit: number): QueryHistoryEntry[];
```

实现：`SELECT * FROM query_history WHERE raw_text LIKE '%keyword%' ORDER BY timestamp DESC LIMIT ?`

### 3.7 REPL 命令处理 (`main.ts`)

| 命令 | 行为 |
|------|------|
| `/history` | 显示最近 10 条查询 |
| `/history <n>` | 显示最近 n 条（上限 50） |
| 空行 | 显示 ≤ 3 条推荐，带序号 |
| 数字序号 | 执行对应推荐（仅刚显示推荐后有效） |

## 4. 数据流

### 4.1 缓存查询流程

```
用户输入 text
  ↓
handleQueryFormatted(text)
  ↓
setBreathLight('thinking')
  ↓
key = QueryCache.buildKey(text, contextMgr.snapshot())
  ↓
QueryCache.get(key) ──hit──► output = cached + ' [cached]'
  │                             ↓
  │ miss                        setBreathLight('idle')
  ↓                             return output
  IntentEngine.parse → CodeIntel.query
  ↓
  output = uiRenderer.render(result)
  ↓
  QueryCache.set(key, output)
  ↓
  setBreathLight('idle')
  ↓
  return output
```

### 4.2 推荐生成流程

```
用户输入空行
  ↓
NodusShell.getRecommendations()
  ↓
RecommendationEngine.generate()
  ├── 1. contextMgr.snapshot().cursor_symbol → 上下文推荐
  ├── 2. store.historyRecent(50) → 高频查询统计
  └── 3. contextMgr.snapshot().recent_queries[0] → 延续推荐
  ↓
  去重，截断为 3 条
  ↓
  TerminalRenderer.renderRecommendations(items)
  ↓
  用户输入序号 → shell.handleQueryFormatted(recommendation.text)
```

## 5. 错误处理

- QueryCache 失效不抛异常，降级为正常查询。
- RecommendationEngine 任何子策略失败不影响其他策略，返回空列表。
- `/history` 查询无结果时显示 "暂无查询历史"。
